--- layout: page title: Events presentation : purpose : name : purpose value :

The event resource provides information about road events (accidents, constructions, special events). The events resource is a list of event elements matching the filtering parameters if any are provided.

method : name : method value : GET parameters : - name : status required : optional description :

By default the API should only send active events. Supported values:

  • active Default value, returns only active events.
  • archived Returns only archived events
  • all Returns both active and archived events.

- name : in_effect_on required : optional description:

Show only events that are active at a specific time, or during a specific time period.

Can be either a single time, or a start and end time joined by a comma. You must provide a complete ISO 8601 datetime, including timezone. So to find, for example, all events active on June 20th (US Eastern Time), you would ask for events between 00:00 and 23:59:
in_effect_on=2013-06-20T00:00-05,2013-06-20T23:59-05
You can also use the special value now to show events currently taking place.

If this argument is not provided, the API will not do any time filtering and will provide all active events, present and future.

- name : severity required : optional description :

Although it is possible to query events for a given severity, it is also possible to query all events below level 2 with severity=<2 or above or equal to level 2 with severity=>=2, which is URL-encoded as severity=%3E%3D2.

- name : bbox required: optional description:

Filter events by geographical bounding box. Four comma-separated coordinates, xmin, ymin, xmax, ymax.

- name : geography required : optional description :

Filtering near a point or linestring. The geography value should be provided as a WKT string in WGS84 latitude/longitude. Example:
...geography=POINT(45.45,-84.85).
The geography must be used in conjuction with the tolerance filter.

- name : tolerance required : conditional description :

Provide a tolerance of radius in meters around if a POINT or LINESTRING is used as filtering parameter. For example ...geography=POINT(45.45,-84.85)&tolerance=50 would retrieve all the events within a circle of 50m radius around the selected point.

- name : jurisdiction required : optional description :

The ID or URL of a jurisdiction, in order to show only events from a given jurisdiction.

Example: jurisdiction=mycity.gov.

- name : event_type required : optional description :

Filter events using the event_type value list.

- name : event_subtype required : optional description :

Filter events using the event_subtype value list.

- name : created required : optional description :

Filter events based on the creation date and time. This filter will mainly be used with a greater than or less than filter created=>2013-05-10T12:00Z.

- name : updated required : optional description :

Filter events based on the last update timestamp. This filter will mainly be used with a greater than or less than filter updated=>2013-05-10T12:00Z.

Note: by default, the servers should only return ACTIVE events. When using the updated filter, in order to get events going from ACTIVE to ARCHIVED, the client must ask all events: ...?status=all

- name : road_name required : optional description :

Filter events based on the name of the road affected by the event.

- name : area_id required : optional description :

Filter events based on the id of an area, based on the area resource (if available).

event : - name: self mandatory: Mandatory type: Link desc:

Self link to the current resource.

- name: jurisdiction type: Link mandatory: Mandatory desc:

Link to the jurisdiction which published the event.

- name: status type: Enum mandatory: Mandatory desc:

Status of the event. The status allows a client to determine if the current event should be considered as currently effective. Value list:

- ACTIVE : The event should be considered as effective
- ARCHIVED : The event should not be considered as effective
- name: headline type: Free text mandatory: Mandatory desc:

Title of the event. Should be shorter than 500 characters.

- name: description type: Free text mandatory: Mandatory desc:

Description of the event. Plain text description of the event, the reason of the event and any other relevant information for travelers.

- name: event_type type: Enum mandatory: Mandatory desc:

Basic type of event. Value list

- CONSTRUCTION :  planned road work
- SPECIAL_EVENT : special events (fair, sport event, etc.)
- INCIDENT : accidents and unplanned road work
- WEATHER_CONDITION : Weather condition affecting the road
- ROAD_CONDITION : Status of the road that might affect travelers.
- name: event_subtypes type: Collection of event_subtype elements mandatory: Optional desc:

List of values to provide more detail about the type of event.

- name: severity type: Enum mandatory: Mandatory desc:

Severity of the event. Value list:

  • 1: Minor, the event has very limited impact on the traffic.
  • 2: Moderate, the event will have a visible impact on the traffic but should not create significant delay and if there is a delay, it should be small and local.
  • 3: Major, the event will have a significant impact on the traffic, probably on a large scale.
  • 9: Undefined. The event has a unknown impact, for example an accident that have been declared without any precise description.

- name: certainty type: Enum mandatory: Optional desc:

Degree of certainty of the event. Should only be used for unplanned events (e.g incidents, weather conditions and pavement conditions events). Value list to be confirmed. Could be observed/likely/possible/unknown.

Value list

- OBSERVED
- LIKELY
- POSSIBLE
- UNKNOWN
- name: created type: DateTime (UTC) mandatory: Mandatory desc:

When the event was initially created.

- name: updated type: DateTime (UTC) mandatory: Mandatory desc:

When the content of the event last changed. Will be the same as created if no updates have occurred. - name: grouped_events type: Collection of links mandatory: Optional desc:

This structure is used to group events together. In specific situations (for example complex construction projects), several events might be related together. This field should be used to point a related event.

- name:

— related

type: Link mandatory: Optional desc:

Link pointing to another event resource related to the current event.

- name: detour type: Free text mandatory: Optional desc:

Description of a detour route to avoid this event.

- name: geography type: Geospatial mandatory: Mandatory desc:

Geographical information about the event. Can be Point, MultiPoint, LineString, MultiLineString, or Polygon.

- name: roads type: Collection of road elements mandatory: Optional desc:

List of roads affected by the current event. One event can impact several roads.

- name: areas type: Collections of area elements mandatory: Optional desc:

Areas affected by the event

- name: schedule type: schedule structure mandatory: Mandatory desc:

Complex item to provide time related information

- name: attachments type: Collection of attachment links mandatory: Optional desc:

Collection of attachments providing additional information about the event (PDFs, images, etc.)

- name:

— related

type: Link mandatory: Optional desc:

Link to an attachment. While only the URL is mandatory, providing the content type, size, and title of the resource (via the type, length, and title attributes in XML) is encouraged.

road : - name: road_name mandatory: Mandatory type: Free text desc:

Name of the road affected by the event. When possible, this field should contain a value that fits the national symbology for roads and highways.

# - name: road_url # mandatory: Optional # type: Link # desc:

If the jurisdiction provides its network, the road_url field provides a link to the road segment that is affected. (TBD)

- name: from type: Free text mandatory: Optional desc:

Approximate start point of the event on the road. It can be an intersection with another street or it can be a mileage indication.

This value should not be used to determine the exact start point since it can be an approximation. The geometry field should be considered as the reference for exact location.

If no to field is provided, the from can be considered as a "near".

- name: to type: Free text mandatory: Optional desc:

Approximate end point of the event on the road. It can be an intersection with another street or it can be a distance indication.

- name: direction type: Enum mandatory: Conditional desc:

Direction of the road that is affected by the event.

If a lane_status is provided, the direction becomes mandatory. In the same situation, if both directions are affected, two occurrences of the road element are needed, one for each direction and with a dedicated lane_status for each direction.

Value list:

- N
- NW
- W
- SW
- S
- SE
- E
- NE
- NONE
- BOTH
- name: lane_status type: bitmap mandatory: Optional desc:

Bitmap representing the lanes that are closed because of the event. For example 0110 means that the 2 central lanes of a 4 lanes roads are closed.

The lane status is applied to direction provided

- name: impacted_systems type: Collection of impacted_system elements mandatory: Optional desc:

Allows provision of information about other systems that can be affected

- name:

— impacted_system

type: Enum mandatory: Optional desc:

Value list of systems:

ROAD
SIDEWALK
BIKELANE
ON_STREET_PARKING
PARKING
- name: restrictions type: Collection of restriction elements mandatory: Optional desc:

Some events may come with some restrictions affecting vehicles using the road (speed, weight).

schedule : - name: timezone mandatory: Optional type: timezone desc:

Timezone to be used for this event, e.g. America/Montreal

- name: start_date type: Date mandatory: Mandatory desc:

Start date of the event.

- name: end_date type: Date mandatory: Optional desc:

End date of the event. If no end date is provided, the event should be considered as continuing indefinitely.

- name: day_of_week type: Enum mandatory: Optional desc:

Space separated list of days of the week where the event should be considered as active. For example Wednesday, Thursday, Friday would be 3 4 5. The days are Monday to Sunday from 1 to 7.

- name: times type: Collection of time mandatory: Optional, multiple occurrences of the time structure are allowed to specify multiple active periods within a day. desc:

Provide detail about the specific times of the event.

- name: specific_dates type: Collection of specific_date mandatory: Optional desc:

Since some days might not follow the regular schedule, it is possible to provide a schedule for individual days (for example vacations).

- name:

— specific_date

type: Custom time format mandatory: Optional desc:

Text string containing the affected date, the start hour and the end hour with the follow format: YYYY-MM-DD HH:mm:ss-HH:mm:ss. If there are multiple disjoint periods on that day, more than one start-end period can be included: YYYY-MM-DD HH:mm:ss-HH:mm:ss HH:mm:ss-HH:mm:ss. If the event is not active that day, no time should be provided: YYYY-MM-DD

time : - name: time type: structure mandatory: Optional desc:

Provide detail about the specific times of the event.

- name:

— start_time

type: Time mandatory: Mandatory in each time element desc:

Start time of the event. Valid for each day on which the event is active.

- name:

— end_time

type: Time mandatory: Mandatory in each time element desc:

End time of the event. Valid for each day on which the event is active.

restriction : - name: restriction_type type: Enum mandatory: Mandatory desc:

Type of restriction that affects vehicles. Value list: 

  • SPEED: Limitation of the speed of vehicles. Unit is in kilometers/hour.
  • WIDTH: Width limitation, mainly for trucks. Unit is meters.
  • HEIGHT: Height limitation, mainly for trucks. Unit is meters
  • WEIGHT: Weight limitation for vehicles. Unit is metric tons.
  • AXLE_WEIGHT: Weight limitation per axle for truck. Unit is metric tons.


    - name: value
      type: float
      mandatory: Mandatory
      desc: 
Value of the limitation. For example a speed limitation of 60km/h will have a value of 60 with a restriction_type set to SPEED
