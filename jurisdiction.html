--- layout: page title: Jurisdiction presentation : a : name : purpose value : The jurisdiction represents any government entity (city, state/province, agency, etc.) publishes open511 data. It is a functional concept used to provide metadata about events such as contact information, etc. b : name : method value : GET representation : - name: self mandatory: Mandatory type: Link desc:

Self link to the current resource. In XML, the rel attribute needs the value self

- name: up type: Link mandatory: Mandatory desc:

Link to the parent resource. In XML, the rel attribute needs the value up

- name: name type: free text mandatory: Mandatory desc:

Name of the jurisdiction.

- name: email type: Free text mandatory: Mandatory desc:

Valid email that can be used to contact the department or person in charge of the data provided by the Open511 API

- name: url type: Link mandatory: Optional desc:

Link pointing to a human readable resource (web page, pdf file, etc) providing details about the Open511 service and the jurisdiction covered.

- name: phone type: Free text mandatory: Optional desc:

Phone number to contact the department or person in charge of the data provided by the Open511 API

- name: description type: Free text mandatory: Optional desc:

Free text that can be used by a client application to provide some information about the Open511 service and the jurisdiction covered.

- name: geography type: Link mandatory: Mandatory desc:

The geography field links an open511 jurisdiction geography resource providing the jurisdiction boundaries.

- name: timezone type: timezone mandatory: Optional desc:

The timezone of the jurisdiction; will be used as the default for events belonging to this jurisdiction. Should always be provided, except in cases where a jurisdiction spans multiple timezones.

- name: license_url type: Link mandatory: Mandatory desc:

Link to a resource containing the license covering the data provided in the API. In XML, the rel attribute needs the value license

- name: languages type: Collection of language mandatory: Optional desc:

List of languages supported by this endpoint. A language element should be provided for each language supported

- name:

— language

type: Enum mandatory: Mandatory desc:

Multiple occurrences supported. The values supported are the same as the language negotiation feature

Parameters and filters

The jurisdiction resource does not support any URL parameter or filter besides format selection and language negotiation.

Example

Example of the XML and JSON representations of the jurisdiction resource

  • XML
  • JSON
<open511
    xmlns:atom="http://www.w3.org/2005/Atom"
    xmlns:gml="http://www.opengis.net/gml"
    xml:lang="en"
    xml:base="http://my.city.gov/open511/"
>  
  <jurisdiction>
    <atom:link rel="self" href="http://my.city.gov/open511/jurisdiction/my-city/" />
    <atom:link rel="up" href="http://my.city.gov/open511/" />    
    <name>My City</name>
    <email>mayor@my.city.gov</email>
    <phone>+1 555-555-0000</phone>
    <description>Official traffic data (construction) from My City</phone>
    <atom:link rel="description" href="http://my.city.gov/open511/presentation.html" />
    <timezone>US/Eastern</timezone>
    <area>
      <gml:Polygon srsName="EPSG:4326">
        <gml:coordinates>-71.17,47.33 -71.15,47.36 -71.10,47.35 -71.20,47.40 -71.17,47.33</gml:coordinates>
      </gml:Polygon>
    </area>
    <atom:link rel="license" href="http://my.city.gov/opendatalicense/"/>
    <languages>
      <language>fr</language>
      <language>en</language>
    </languages>
  </jurisdiction>
</open511>
