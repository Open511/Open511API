--- layout: page title: Jurisdiction presentation : a : name : purpose value : The jurisdiction represents any government entity (city, state/province, agency, etc.) publishes open511 data. It is a functional concept used to provide metadata about events such as contact information, etc. b : name : method value : GET representation : a : name: self mandatory: Mandatory type: Link desc:

Self link to the current resource. In XML, the rel attribute needs the value self

b : name: up type: Link mandatory: Mandatory desc:

Link to the parent resource. In XML, the rel attribute needs the value up

c : name: name type: free text mandatory: Mandatory desc:

Name of the jurisdiction.

d : name: email type: Free text mandatory: Mandatory desc:

Valid email that can be used to contact the department or person in charge of the data provided by the Open511 API

e : name: url type: Link mandatory: Optional desc:

Link pointing to a human readable resource (web page, pdf file, etc) providing details about the Open511 service and the jurisdiction covered.

f : name: phone type: Free text mandatory: Optional desc:

Phone number to contact the department or person in charge of the data provided by the Open511 API

g : name: description type: Free text mandatory: Optional desc:

Free text that can be used by a client application to provide some information about the Open511 service and the jurisdiction covered.

h : name: area type: geography field mandatory: Mandatory Issue 14 desc:

Geographical coverage of the jurisdiction. This field only accepts a polygon or multipolygons.

i : name: license type: License structure mandatory: Mandatory desc:

Provides information concerning the license covering the data provided by the API. Issue 6

k : name: languages type: Collection of language mandatory: Optional desc:

List of languages supported by this endpoint. A language element should be provided for each language supported

m : name:

— language

type: Enum mandatory: Mandatory desc:

Multiple occurrences supported. The values supported are the same as the language negotiation feature

license : a : name: url type: Link mandatory: Mandatory desc:

Direct link to the license covering the data provided by the API. In XML, the rel attribute needs the value license

b : name: licenseName type: Enum mandatory: Optional desc:

The name of one of the well-known licenses and part of the enum supported by open511

Enum:

- ODBL
- CC0
- CC-BY
- ...
c : name: licenseConditions type: Collection of condition mandatory: Optional desc:

List of languages supported by this endpoint. A language element should be provided for each language supported

d : name:

— licenseCondition

type: Enum mandatory: At least one element mandatory if the licenseConditions structure is present desc:

Generic conditions that are contained in the license.

Enum:

- ATTRIBUTION
- SHAREALIKE
- ...

Parameters and filters

The jurisdiction resource does not support any URL parameter or filter besides format selection and language negotiation.

Example

Example of the XML and JSON representations of the jurisdiction resource

  • XML
  • JSON
<open511
    xmlns:atom="http://www.w3.org/2005/Atom"
    xmlns:gml="http://www.opengis.net/gml"
    xml:lang="en"
    xml:base="http://my.city.gov/open511/"
>  
  <jurisdiction>
    <atom:link rel="self" href="http://my.city.gov/open511/jurisdiction/my-city/" />
    <atom:link rel="up" href="http://my.city.gov/open511/" />    
    <name>My City</name>
    <email>mayor@my.city.gov</email>
    <phone>+1 555-555-0000</phone>
    <description>Official traffic data (construction) from My City</phone>
    <atom:link rel="description" href="http://my.city.gov/open511/presentation.html" />
    <area>
      <gml:Polygon srsName="EPSG:4326">
        <gml:coordinates>-71.17,47.33 -71.15,47.36 -71.10,47.35 -71.20,47.40 -71.17,47.33</gml:coordinates>
      </gml:Polygon>
    </area>
    <license>
      <atom:link rel="license" href="http://my.city.gov/opendatalicense/"/>
      <licenseName>CC-BY</licenseName>
      <conditions>
        <condition>ATTRIBUTION</condition>
        <condition>SHAREALIKE</condition>
      </conditions>
    </license>
    <languages>
      <language>fr</language>
      <language>en</language>
    </languages>
  </jurisdiction>
</open511>
