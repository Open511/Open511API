Reports are events reported by citizens/untrusted sources, it is the crowdsource feature of the Open511 resource.

The POST method should be used in conjunction with the x-www-form format to submit new crowdsourced reports.

The GET methode will return a list of all the active reports

Report submission can only be sent to the orignal endpoint (not to an agregator) method : name : method value : POST, GET parameters : - name : status required : optional description :

By default the API should only send active events. Supported values:

  • active Default value, returns only active events.
  • archived Returns only archived events
  • all Returns both active and archived events.

- name : bbox required: optional description:

Filter events by geographical bounding box. Four comma-separated coordinates, xmin, ymin, xmax, ymax.

postrepresentation : - name: jurisdiction mandatory: Mandatory type: Free text desc:

Full url of the jurisdiction linked with the report.

- name: lat type: float mandatory: Mandatory desc:

Latitude of the report in WGS84 projection

- name: long type: float mandatory: Mandatory desc:

Longitude of the report in WGS84 projection

- name: first_name type: free text mandatory: Optional desc:

First name of the person submitting the report

- name: last_name type: free text mandatory: Optional desc:

Last name of the person submitting the report

- name: email type: free text mandatory: Mandatory desc:

Email of the person submitting the report

- name: phone type: free text mandatory: Optional desc:

Phone number of the person who submitted the event

- name: device_id mandatory: Mandatory type: Free text desc:

Unique identifier for the device used to submit the report. It can be the MAC address of the network interface or use the getDeviceId() of Android devices or [UIDevice currentDevice].uniqueIdentifier for iPhone device

- name: description type: Free text mandatory: Mandatory desc:

Description provided by the person who submitted the report.

- name: event_type type: Enum mandatory: Mandatory desc:

Type of event declared by the person who submitted the report. Same value list as the event.event_type field.

- name: severity type: Enum mandatory: Optional desc:

Severity of the event as judged by the person who submitted the report. This field use the same enumeration as the severity field of the event resource

- name: media type: binary mandatory: Optional desc:

Inline media data related to the event.

getrepresentation : - name: juridiction type: free text mandatory: Mandatory desc:

Jurisdiction related to the report.

- name: geography type: geography field mandatory: Mandatory desc:

Location of the report. Only POINT is supported.

- name: description type: Free text mandatory: Mandatory desc:

Description provided by the submitter of the report.

- name: event_type type: Enum mandatory: Optional desc:

Event type as provided by the submitter of the report. Same value list as the event.event_type list.

- name: severity type: Enum mandatory: Optional desc:

Severity as provided by the submitter of the report. Same value list as the event.severity - name: status type: Enum mandatory: Mandatory desc:

Status attributed by the jurisdiction. Value list:

  • OPEN - Default value. The report is active but not confirmed by the jurisdiction
  • CONFIRMED - The report is active and the jurisdiction has confirmed the report.
  • REJECTED - The jurisdiction has rejected the report
  • CLOSED - The report has been closed (either after being confirmed or not)
  • DUPLICATED - The report is considered as a duplicated of an existing event/report.

    - name: creation_date type: DateTime mandatory: Mandatory desc:

    Creation date as defined by the jurisdiction.

    - name: last_update type: DateTime mandatory: Mandatory desc:

    Comment provided by the jurisdiction

    - name: related_url type: Link mandatory: Optional desc:

    Link pointing a related resource. As a general rule, the related link should used to point an official event that would replace the report.

    POST a report

    In order to post a report, the client has to send values with x-www-form (if not image is provided) or multiple/form-data (if an image is provided) content types.

    The URL to use for the post is the one defined in the discovery resource for the report service (the same to send GET requests)

    POST data structure

    Example of report POST with content-type x-www-form.

    POST /open511/reports/
Host: api.city.gov
Content-Type: application/x-www-form-urlencoded; charset=utf-8

jurisdiction=city.gov&lat=37.76524078&long=-122.4212043&device_id=tt222111&
first_name=John&last_name=Smith&phone=111111111&description=Major+accident+on+highway+680&
event_type=ACCIDENT&severity=2
